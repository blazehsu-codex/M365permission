ChatGPT Apps 寫入權限分級指南

Microsoft 365 — Outlook Calendar・Outlook Mail・SharePoint・Teams　｜　點擊功能名稱可查看詳細說明

一、風險等級定義

Level 1　低風險

草稿 / 個人狀態變更

不直接對外發送、不刪除、不影響共享資源；操作多半可回復。

Level 2　中低風險

新增內容 / 建立結構

建立事件、資料夾、聊天、任務、頻道；通常不會立即造成外部影響。

Level 3　中高風險

改動既有內容 / 對外送出

發信、回覆、移動、更新、建立分享連結；容易造成誤寄或排程混亂。

Level 4　高風險

刪除 / 共享帳號 / 批次操作

刪除、取消會議、共享信箱寫入、批次搬移；影響面廣且難以回復。

三、什麼情況開什麼等級

方案 A

只開 Level 1

試點導入初期・使用者對 AI 尚在建立信任
Level 1
  • Outlook Mail：草稿、分類、已讀未讀
  • 不開任何直接送出、刪除、共享、批次操作
  • 讓 ChatGPT 協助整理與起草，人工確認後再手動送出
方案 B

開到 Level 2

內部個人效率場景・希望 AI 能幫建立工作內容
Level 1Level 2
  • Calendar：建立事件、RSVP 回覆
  • Mail：草稿 + 建立信箱資料夾
  • SharePoint：建立資料夾
  • Teams：建立 chat、channel、planner task
不開共享信箱 / 共享行事曆・不開直接送信・不開刪除與批次搬移
方案 C

開到 Level 3

使用者已接受 AI 代理日常操作・有明確 SOP 或人工確認步驟
Level 1Level 2Level 3
  • Mail：send_email、reply_to_email、forward_email、schedule_email
  • Calendar：update_event
  • SharePoint：上傳、更新、搬移、分享連結
  • Teams：發訊、回覆、任務更新
發送前確認收件人/標題/內容・檔案更新前確認目標路徑・分享連結限制內部人員
方案 D

開到 Level 4

行政助理、PMO、客服共用信箱管理者、知識庫管理者
Level 1Level 2Level 3Level 4
  • 共享信箱 / 共享行事曆寫入
  • 刪除或取消操作
  • SharePoint 批次搬移、批次建資料夾、刪除
必要前提：僅限少數角色・二次確認機制・稽核紀錄・先於測試租戶或低敏感區域驗證

六、核心原則總結

  1. 預設只開 Level 1–2，讓 ChatGPT 幫助整理與起草，人工最終確認後再送出。
  2. Level 3 採「有明確需求再開」策略，開放前要確認有 SOP 或二次確認流程。
  3. Level 4 不做預設開放，改為角色型授權（行政助理、PMO、知識庫管理員等）。
  4. Shared mailbox / shared calendar 一律比個人帳號高一個等級看待。
  5. SharePoint 的分享與批次操作，應比一般訊息或會議操作採更嚴格的管控標準。
